THIRD-PARTY SOFTWARE NOTICES AND INFORMATION

The CodeBrix.Audio.MitLicenseForever NuGet package ships two assemblies:

  * CodeBrix.Audio        — a fully managed, cross-platform audio FILE library
                            (WAV/MP3 reading, Standard MIDI File read/write,
                            MP3 ID3v2 tags, DSP primitives). Contains no native
                            code. Adapted from NAudio and NLayer (entries 1-2).
  * CodeBrix.Audio.Engine — a cross-platform audio ENGINE (playback, recording,
                            effects, editing/mixing, MIDI, synthesis,
                            visualization) with a bundled native miniaudio
                            backend for Windows, macOS, and Linux. Vendored from
                            SoundFlow and miniaudio (entries 3-4).

Both assemblies incorporate source code (and, for the Engine, a native library)
adapted from the open-source projects listed below. Every incorporated project
is distributed under a license — MIT, or the public-domain Unlicense / MIT-0 —
that is permissive and compatible with the MIT License of CodeBrix.Audio itself.
The original copyright and permission notices are reproduced in full below, as
those licenses require them to travel with the software.

For every incorporated SOURCE file, the C# `namespace` line carries a provenance
comment of the form `//was previously: <original.namespace>` recording the
upstream namespace it was adapted from. The exact modifications differ per
component (namespaces are always renamed to a `CodeBrix.Audio.*` hierarchy; other
changes — conditional-compilation stripping, nullable-annotation changes,
doc-comment additions — are made only where noted). See the "Modifications" note
in each entry below for specifics.

1. NAudio
================================================================================

Project:    NAudio
Source URL: https://github.com/naudio/NAudio
Version:    3.0.0 (pre-release / "preview" line; main branch as of 2026-06)
Authors:    Mark Heath and NAudio contributors
License:    MIT

Incorporated source (production):
  Adapted from the NAudio.Core and NAudio.Midi projects. The portions brought
  into CodeBrix.Audio are limited to the fully managed, cross-platform file
  handling, codec, DSP, and MIDI code:
    - WAV reading/writing and the wave-stream / wave-format / sample-provider
      / wave-provider plumbing (NAudio.Core/Wave/*).
    - MP3 frame parsing and the managed MP3 file-reader plumbing, including
      ID3v2 tag reading (NAudio.Core/FileFormats/Mp3/*).
    - G.711 A-law / mu-law (and related) codecs required by the WAV read path
      (NAudio.Core/Codecs/*).
    - DSP primitives for analysis: FFT, biquad filters, envelope follower,
      voice-activity detection, and supporting math/utility types
      (NAudio.Core/Dsp/*, NAudio.Core/Utils/*).
    - Standard MIDI File reading and writing: MidiFile, the MidiEvent type
      hierarchy, and MidiEventCollection (NAudio.Midi/*).

Incorporated source (tests):
    The test project under tests/CodeBrix.Audio.Tests contains test files and
    test helpers adapted from the NAudio.Core.Tests project (MIT). They were
    converted from NUnit to xUnit v3 + SilverAssertions and re-namespaced to
    CodeBrix.Audio.Tests.*. Test code is not redistributed in the NuGet package.

Modifications made during the port:
    - Namespaces renamed from NAudio.* to CodeBrix.Audio.* (see provenance
      comments on each file's namespace line).
    - Windows-only and other platform-specific NAudio projects (NAudio.WinMM,
      NAudio.Wasapi, NAudio.Asio, NAudio.Dmo, NAudio.Alsa, NAudio.WinForms,
      etc.) were NOT incorporated; CodeBrix.Audio contains no audio-device
      playback or recording code.
    - NAudio's ACM/DMO-based MP3 decoding (Windows-only) was NOT incorporated;
      MP3 decoding is provided by the managed NLayer decoder (see entry 2).
    - NAudio.Core file-format parsers and engines unrelated to the supported
      feature set (SoundFont/SFZ parsing, the sampler/synthesis engine,
      sequencing, and effects) were NOT incorporated.
    - Framework-selection conditional-compilation branches that do not execute
      on .NET 10 were stripped; nullable-reference-type annotations removed;
      file-scoped namespaces applied; XML doc comments added as needed.

3. SoundFlow  (vendored as CodeBrix.Audio.Engine)
================================================================================

Project:    SoundFlow
Source URL: https://github.com/LSXPrime/SoundFlow
Version:    1.4.1 (release/v1.4.1, commit a0d1be1)
Authors:    LSXPrime and SoundFlow contributors
License:    MIT

Incorporated source (production):
    SoundFlow's entire core library (its `Src` project) was vendored verbatim to
    form the CodeBrix.Audio.Engine assembly: the cross-platform audio engine and
    its miniaudio backend, audio components and providers, modifiers/effects, the
    editing/mixing engine, metadata (audio-tag, MIDI, and SoundFont reading and
    writing), the MIDI stack, the synthesis / SoundFont engine, the security
    features (watermarking, encryption, fingerprinting), and visualization.

    SoundFlow's OPTIONAL add-on projects were NOT incorporated: the WebRTC audio
    processing extension (Extensions/SoundFlow.Extensions.WebRtc.Apm), the FFmpeg
    codec (Codecs/SoundFlow.Codecs.FFMpeg), and the PortMidi backend
    (Midi/SoundFlow.Midi.PortMidi), together with their native libraries. The
    ONLY native dependency carried into CodeBrix.Audio.Engine is the miniaudio
    backend (see entry 4).

Modifications made during the port:
    - Namespaces renamed from SoundFlow.* to CodeBrix.Audio.Engine.* (see the
      `//was previously:` provenance comment on each file's namespace line).
    - Target framework moved from net8.0 to net10.0. SoundFlow's other project
      settings were kept as-is (nullable reference types enabled, implicit usings,
      unsafe blocks, trim/AOT analyzers) so future re-syncs from upstream stay
      mechanical: NO nullable annotations were stripped and NO doc comments were
      added (unlike the NAudio/NLayer port in entries 1-2).
    - The native backend library was renamed from "miniaudio" to
      "codebrix_miniaudio" — the P/Invoke library name in Backends/MiniAudio/
      Native.cs and the CMake LIBRARY_NAME — so the shipped binary never clashes
      with a stock miniaudio.* that a consuming application might also load.
    - SoundFlow's native build inputs (its Native/miniaudio-backend C wrapper and
      CMake project) were vendored under native/miniaudio and pruned to the
      platforms we ship (Windows, macOS, Linux). The six shipped RIDs are win-x64,
      win-arm64, linux-x64, linux-arm64, osx-x64, osx-arm64.

Native binaries currently shipped:
    The codebrix_miniaudio.{dll,so,dylib} binaries committed under
    src/CodeBrix.Audio.Engine/Backends/MiniAudio/runtimes/ are SoundFlow v1.4.1's
    own prebuilt miniaudio libraries, renamed. They are a stopgap; the sources in
    native/miniaudio allow rebuilding equivalents from scratch (see entry 4 and
    native/miniaudio/README.txt).
